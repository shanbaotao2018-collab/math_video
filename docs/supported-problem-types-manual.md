# 支持题型使用手册

本文档面向调用方和内容生产者，说明当前代数视频生成系统可以稳定处理哪些题型、推荐怎么输入、会返回什么结果，以及哪些边界会进入 fallback / unsupported。

当前系统不是通用 CAS。它的目标是：对支持范围内的初中代数题，生成可用于短视频讲解的结构化推导、字幕、强调点、节奏点和渲染数据。

## 1. 快速使用

### 1.1 命令行生成产品 JSON

```bash
npm run product -- "2x+3=7"
```

常用示例：

```bash
npm run product -- "2x+3=7"
npm run product -- "-2x+3>7"
npm run product -- "x/2+x/3=5"
npm run product -- "(x-1)/(x+2)>0"
npm run product -- "x+y=5, x-y=1"
npm run product -- "x^2-2x-1=0"
```

### 1.2 生成 HTML 预览和视频资产

```bash
npm run video -- "2x+3=7" --html-only
```

会输出到 `out/`，包括：

- preview HTML
- timeline HTML / screenshot
- render plan
- subtitle cues
- voice cues
- publishing pack
- product entry JSON

### 1.3 Delivery API

启动服务：

```bash
npm run delivery
```

查看能力目录：

```bash
curl -s http://127.0.0.1:4173/capabilities
```

求解并生成产品数据：

```bash
curl -s http://127.0.0.1:4173/solve \
  -H 'Content-Type: application/json' \
  -d '{"equation":"x+y=5, x-y=1","includeLesson":true,"presentationMode":"auto"}'
```

## 2. 返回结果怎么看

`npm run product` 和 `POST /solve` 返回统一 product entry。调用方重点看这些字段：

| 字段 | 含义 |
| --- | --- |
| `supported` | 是否进入当前支持链路 |
| `quality` | `full / partial / fallback / unsupported` |
| `qualityTier` | `instant / basic / standard / detailed` |
| `family.id` | 识别出的题型家族 |
| `presentationStrategy.id` | 实际展示策略 |
| `problem.steps` | 数学步骤 |
| `teachingScript` | 讲解脚本文案 |
| `subtitleCuePlan` | 字幕时间轴 |
| `emphasisPlan` | hook / 易错点 / 结果强调 |
| `rhythmPlan` | 节奏、打断、停顿 |
| `lesson` | 可直接送入当前 Remotion scene 的渲染数据 |
| `videoRender.renderable` | 是否具备视频渲染条件 |

`quality` 的含义：

| quality | 含义 |
| --- | --- |
| `full` | 主链路和增强层完整可用 |
| `partial` | 主链路可用，增强层部分降级，通常仍可出片 |
| `fallback` | 题型暂不支持，但返回最小可渲染说明页 |
| `unsupported` | 明确不支持，通常不返回可渲染 problem |

## 3. 当前支持题型总览

| 题型 family | 中文名 | 推荐展示 | 结果形态 | 示例 |
| --- | --- | --- | --- | --- |
| `linear_equation` | 一元一次方程 | `compact_steps` | `x=...` | `2x=8`, `2x+3=7` |
| `linear_inequality` | 一元一次不等式 | `compact_steps` | `x>...` / `x<=...` | `2x+3>7`, `-2x+3>7` |
| `fraction_equation` | 分式方程 | `full_steps` | `x=...` | `x/2+3=7`, `x/2+x/3=5` |
| `fraction_inequality` | 分式不等式 | `semantic_full_steps` | 区间 / 并集 / 不等式解集 | `(x+1)/2<=4`, `(x-1)/(x+2)>0` |
| `linear_system` | 二元一次方程组 | `semantic_full_steps` | 有序数对 / 无解 / 无穷多解 | `x+y=5, x-y=1` |
| `quadratic_equation` | 一元二次方程 | `full_steps` | 两个实根 / 重根 / 无实数解 | `x^2-5x+6=0`, `x^2-2x-1=0` |

## 4. 详细支持范围

### 4.1 一元一次方程

适合做短平快讲解，默认推荐 `compact_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 一步一次方程 | `2x=8` | 通常归为 `instant`，可只展示答案或极简步骤 |
| 带常数项 | `2x+3=7` | 保留移项、化简、答案 |
| 两边含未知数 | `2x+3=x+7` | 支持把未知数项整理到同一侧 |
| 展开括号 | `2(x+3)=14` | 支持先展开再继续一次方程链路 |
| 多括号展开 | `2(x+1)+3(x+2)=13` | 支持多个一次括号的整理 |

结果示例：

```text
x=4
x=2
x=-2
```

当前边界：

- 小数系数暂不作为稳定支持范围，例如 `2.5x+3=7`。
- 默认只面向单变量 `x` 的一次方程链路。

### 4.2 一元一次不等式

适合突出“变号”易错点，默认推荐 `compact_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 基础一次不等式 | `2x+3>7` | 支持移项、化简、同除 |
| 负系数变号 | `-2x+3>7` | 同除负数时会保留不等号反向语义 |
| 含括号不等式 | `2(x+3)<=14` | 支持展开后求解 |

结果示例：

```text
x>2
x<-2
x<=4
```

当前边界：

- 不支持含额外变量的目标式，例如 `2x+3>=y`。
- 当前稳定范围是单变量 `x` 的一次不等式。

### 4.3 分式方程

适合展示“先去分母”的方法，默认推荐 `full_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 单分式 + 常数 | `x/2+3=7` | 支持去分母后转一次方程 |
| 括号分子 | `(x+1)/2=4` | 支持括号分子整体处理 |
| 多分式求和 | `x/2+x/3=5` | 支持通分 / 去分母 / 合并同类项 |

结果示例：

```text
x=8
x=7
x=6
```

当前边界：

- 负分母分式方程暂不稳定支持，例如 `x/(-2)+3=7`。
- 更复杂的嵌套分式、变量分母方程暂不作为稳定支持范围。

### 4.4 分式不等式

适合详细讲解定义域、临界点和区间符号，默认推荐 `semantic_full_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 分式不等式基础形态 | `(x+1)/2<=4` | 分母为常数时可转一次不等式并结合限制 |
| 分式 + 常数不等式 | `x/2+3>=7` | 支持去分母后求解 |
| 变量分母单临界点 | `1/(x+2)>0` | 支持定义域和符号判断 |
| 标准有理不等式 | `(x-1)/(x+2)>0` | 支持临界点、区间分析和并集结果 |
| 变量分母与右侧比较 | `x/(x+2)<=1` | 支持整理后分析定义域与解集 |

结果示例：

```text
x<=7
x>2
(-\infty,-2)\cup(1,+\infty)
```

当前边界：

- 暂不支持复杂二次分子 / 多因式分子，例如 `(x^2-1)/(x+2)>0`。
- 更通用的高阶有理不等式不在当前稳定范围内。

### 4.5 二元一次方程组

适合展示代入、消元、回代和结果分类，默认推荐 `semantic_full_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 代入法 | `x+y=5, x-y=1` | 支持改写、代入、求一个变量、回代 |
| 已解出变量的代入法 | `y=2x+1, x+y=7` | 支持直接代入另一条方程 |
| 消元法 | `2x+y=7, 2x-y=1` | 支持两式相加 / 相减消元 |
| 无解分类 | `x+y=2, x+y=5` | 支持判断条件矛盾 |
| 无穷多解分类 | `x+y=2, 2x+2y=4` | 支持判断同一个条件 |

结果示例：

```text
(x,y)=(3,2)
(x,y)=(2,3)
无解
无穷多解
```

输入格式建议：

```text
x+y=5, x-y=1
x+y=5; x-y=1
```

中文逗号和分号会做归一化，但建议生产环境统一用英文逗号。

当前边界：

- 不支持三元方程组，例如 `x+y+z=3, x-y=1`。
- 不支持非线性方程组、复杂含分母方程组。

### 4.6 一元二次方程

适合标准流程讲解，默认推荐 `full_steps`。

支持形态：

| 形态 | 示例 | 说明 |
| --- | --- | --- |
| 开平方链路 | `x^2=16` | 支持正负两个分支 |
| 平方式链路 | `(x+1)^2=9` | 支持开平方后求解 |
| 可因式分解标准型 | `x^2-5x+6=0` | 支持因式分解和零乘积法 |
| 已因式分解型 | `(x-2)(x-3)=0` | 支持拆成两个一次方程 |
| 判别式 / 求根公式，两实根 | `x^2-2x-1=0` | 支持 Δ 判断和公式法 |
| 判别式 / 求根公式，无实根 | `x^2+2x+5=0` | 支持返回无实数解 |

结果示例：

```text
x=-4 或 x=4
x=2 或 x=3
x=1-\sqrt{2} 或 x=1+\sqrt{2}
无实数解
```

当前边界：

- 只支持单变量 `x` 的二次方程。
- 不支持含 `xy`、多变量二次式，例如 `x^2+2xy+1=0`。
- 不支持二次不等式。
- 复杂前置变形、含分母或根式的一般二次方程暂不作为稳定支持范围。

## 5. 输入规范

建议输入：

- 使用变量 `x`；方程组可使用 `x` 和 `y`。
- 使用英文运算符：`+ - * / ^ = < > <= >=`。
- 方程组用英文逗号分隔：`x+y=5, x-y=1`。
- 小于等于 / 大于等于可输入 `<=` / `>=`，也可输入 `≤` / `≥`。
- 可以带简单中文前缀，例如 `解方程 x^2-2x-1=0`，系统会尝试归一化。

不建议输入：

- 小数系数：`2.5x+3=7`
- 三元或多变量表达式：`x+y+z=3`
- 非代数题、几何题、函数图像题
- 高阶方程：`x^3-1=0`
- 二次不等式：`x^2-5x+6>0`
- 复杂嵌套分式或根式方程

## 6. 展示策略

默认 `presentationMode` 为 `auto`。系统会根据题型复杂度自动选择：

| mode | 适用场景 |
| --- | --- |
| `answer_only` | 只看最终答案或极简步骤 |
| `compact_steps` | 简单线性题，保留关键推导 |
| `full_steps` | 标准流程题，如分式方程、一元二次方程 |
| `semantic_full_steps` | 需要定义域、区间、分支、方程组关系的复杂链路 |

可手动覆盖：

```bash
npm run product -- --presentation-mode answer_only "2x+3=7"
npm run product -- --presentation-mode full_steps "2x+3=7"
npm run product -- --presentation-mode semantic_full_steps "x+y=5, x-y=1"
```

## 7. Fallback / Unsupported 行为

默认行为是 `fallbackOnUnsupported=true`：

- 不支持题型会返回 `supported: false`。
- `quality` 通常为 `fallback`。
- 仍会返回最小可渲染 problem，用于提示“当前题型暂不支持”。

关闭 fallback：

```bash
npm run product -- --no-fallback "x/(-2)+3=7"
```

此时通常返回：

```json
{
  "supported": false,
  "quality": "unsupported"
}
```

常见不支持示例：

| 输入 | 原因 |
| --- | --- |
| `2.5x+3=7` | 小数系数一次方程暂不稳定支持 |
| `2x+3>=y` | 含额外变量 |
| `x/(-2)+3=7` | 负分母分式方程暂不稳定支持 |
| `(x^2-1)/(x+2)>0` | 复杂二次分子分式不等式 |
| `x+y+z=3, x-y=1` | 三元方程组 |
| `x^2+2xy+1=0` | 多变量二次方程 |

## 8. 推荐验收命令

修改题型支持、文案或渲染策略后，建议至少跑：

```bash
npm run typecheck
npm run evaluate
npm run video -- "2x+3=7" --html-only
```

如果只想看某一题的完整结构：

```bash
npm run product -- "x^2-2x-1=0"
```

## 9. 当前最适合投产的题

优先选择这些输入做批量内容：

```text
2x+3=7
2(x+3)=14
-2x+3>7
x/2+x/3=5
(x-1)/(x+2)>0
x+y=5, x-y=1
2x+y=7, 2x-y=1
x^2-5x+6=0
x^2=16
x^2-2x-1=0
```

这些题能覆盖当前最核心的短视频能力：移项、变号、去分母、定义域、区间分析、方程组、二次方程分支和结果页强化。
